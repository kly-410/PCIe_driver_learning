# S1_02_实验 — 链路层与 Flow Control + DOE

> 本节实验较难直接接触硬件层，重点在通过工具观察链路状态和理解 Flow Control 概念。

---

## 实验 1：链路状态深度分析

**目标**：通过 `lspci -vv` 读取链路详细信息，理解 Flow Control 协商结果。

**步骤**：
```bash
### 读取 RC（00:00.0）的链路状态
sudo lspci -vv -nn -s 00:00.0 | grep -E "LnkSta|LnkCtl|FC"

### 读取第一个 EP 的链路状态
sudo lspci -vv -nn -s 01:00.0 | grep -E "LnkSta|LnkCtl|FC"

### 完整输出（方便整体分析）
sudo lspci -vv -nn -s 00:00.0
sudo lspci -vv -nn -s 01:00.0
```

**预期结果**：
- `LnkSta`：显示当前协商速率（Gen1/2/3）和宽度（x1/x2/x4）
- `LnkCtl`：显示 Flow Control 状态
- 能看到 RC 和 EP 的协商结果是否一致

**验收标准**：
- [ ] 能读出 RC 和 EP 的链路状态
- [ ] 能解释为什么两个设备协商结果应该一致

## 实验 2：PCIe AER 错误状态读取

**目标**：通过 AER（Advanced Error Reporting）读取链路错误，理解数据链路层的错误检测机制。

**步骤**：
```bash
### 检查 AER 状态寄存器
sudo lspci -vv -nn -s 00:00.0 | grep -E "Uncorrectable Error|Correctable Error|AER"

### 检查 AER Uncorrectable Error Status
sudo setpci -s 00:00.0 AER_UE_STATUS.w

### 检查 AER Correctable Error Status
sudo setpci -s 00:00.0 AER_CE_STATUS.w
```

> 注：AER 寄存器地址需要通过 PCIe Capability 结构找到偏移量。

**预期结果**：正常情况下所有错误状态位为 0（无错误）

**验收标准**：
- [ ] 能理解 AER 错误状态位的含义
- [ ] 确认当前链路无错误（所有状态位为 0）

### 改错题（Challenge）

代码中故意埋了一个 bug，运行或审查后找出问题。

**实验代码**（`S1_02_doe.c`，在 `03_S1_链路层与FlowControl_DOE/` 目录下）：

下面的代码构造了一个 DOE 数据对象写入请求（DOE Write），用来写一个设备的 DOE 配置空间：

```c
/* S1_02_doe.c 片段 —— 故意埋了 DOE 描述符格式 bug */
#include <stdio.h>
#include <stdint.h>

/* DOE 描述符格式（来自 DOE Spec r1.0）
 * DW0 布局：
 *   [15:0]  = Object ID
 *   [31:16] = Vendor ID（标准对象为 0x0000）
 * DW0 额外字段：
 *   W       = bit[5]（1=写）
 *   LengthDW = bit[4:0]（数据长度，单位 DW）
 * DW1 = 数据（如果 W=1）
 */

struct doe_descriptor {
    uint32_t dw0;   /* Object ID + Vendor ID + Flags */
    uint32_t dw1;   /* 数据 */
};

/* 构造 DOE Write 描述符 */
static inline uint32_t make_doe_write(uint16_t object_id, uint32_t data)
{
    uint32_t desc = 0;
    desc |= (object_id << 16);   /* Object ID → [31:16]（错！）*/
    desc |= (1 << 6);            /* W=1 → bit[6]（错！）*/
    desc |= (1 & 0x1F);          /* LengthDW=1 → bit[0]（错！）*/
    return desc;
}

void write_doe_object(uintptr_t doe_base, uint16_t object_id, uint32_t wdata)
{
    uint32_t desc = make_doe_write(object_id, wdata);
    /* 写 DOE 邮箱寄存器（数据链路层，不走 TLP）*/
    *(volatile uint32_t *)(doe_base + 0x00) = desc;
    *(volatile uint32_t *)(doe_base + 0x04) = wdata;
}

int main(void)
{
    /* 假设 DOE 邮箱基址 = 0xf7c00000 */
    write_doe_object(0xf7c00000, 0x0001, 0x12345678);
    return 0;
}
```

**问题**：DOE 描述符的 DW0 bit 字段排布和 DOE Spec 不符：

1. Object ID 应该放在 [15:0]，代码放到了 [31:16]
2. W bit 是 bit[5]，代码放到了 bit[6]
3. LengthDW 在 [4:0]，代码放到了 bit[0]

1. 找出哪一行的 bit 字段排布和 DOE Spec 不符
2. 写出正确的 `make_doe_write` 函数实现

> 提示：DOE Spec r1.0 Section 3.1 定义了 DOE 描述符的 DW0 布局。Object ID 在低 16 位，W 在 bit[5]，LengthDW 在 bit[4:0]。

预期答案：`make_doe_write` 应改为：
```c
static inline uint32_t make_doe_write(uint16_t object_id, uint32_t data)
{
    return object_id | (1 << 5) | (1 << 0);  /* Object ID 在 [15:0]，W 在 bit[5]，Length 在 bit[4:0] */
}
```
当前代码的三处错误：Object ID 左移 16 位（应为 0），W 在 bit[6]（应为 bit[5]），Length 在 bit[0]（应为 [4:0] 位）。
