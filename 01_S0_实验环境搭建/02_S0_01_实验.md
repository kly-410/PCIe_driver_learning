# S0_01_实验 — 实验环境搭建

> 本节课为纯环境验证，共 3 个实验，全部在宿主机执行，不需要进入 QEMU Guest。

---

## 实验 1：QEMU 版本验证

**目标**：确认 QEMU 9.0 已正确安装

**步骤**：
```bash
# 检查 QEMU 版本
/usr/local/bin/qemu-system-x86_64 --version
# 预期输出：QEMU emulator version 9.0.0

# 确认 qemu-system-x86_64 在 PATH 中（可选）
export PATH="/opt/qemu/bin:$PATH"
qemu-system-x86_64 --version
```

**预期结果**：输出 QEMU 9.0.x，版本号 ≥ 9.0.0

**验收标准**：
- [ ] `qemu-system-x86_64 --version` 输出包含 "9.0"
- [ ] `which qemu-system-x86_64` 能找到路径

---

## 实验 2：开发工具链验证

**目标**：确认 GCC / GDB / Linux headers 完整

**步骤**：
```bash
# GCC
gcc --version | head -1
# 预期：gcc (Ubuntu 13.2.0-23ubuntu4) 13.2.0

# GDB Python 支持（调试内核模块必需）
gdb -batch -ex "python print('GDB python OK')"
# 预期：GDB python OK

# Make
make --version | head -1
# 预期：GNU Make 4.3

# Linux headers
ls /lib/modules/$(uname -r)/build/Makefile 2>/dev/null && echo "headers OK"
# 预期：headers OK
```

**预期结果**：所有工具返回版本号或 OK

**验收标准**：
- [ ] GCC 版本 ≥ 13
- [ ] GDB Python 扩展可用
- [ ] Make 可用
- [ ] linux-headers Makefile 存在

## 实验 3：PCI 子系统配置验证

**目标**：确认内核 PCI 配置完整

**步骤**：
```bash
# 检查关键 PCI 配置选项
grep -E "CONFIG_PCI=|CONFIG_PCI_MSI=|CONFIG_PCIEAER=|CONFIG_PCIEPORTBUS=" \
    /boot/config-$(uname -r)

# 预期（均为 =y 或 =m）：
# CONFIG_PCI=y
# CONFIG_PCI_MSI=y
# CONFIG_PCIEPORTBUS=y
# CONFIG_PCIEAER=y
```

**预期结果**：所有关键配置为 y 或 m，无 =n

**验收标准**：
- [ ] CONFIG_PCI=y 或 =m
- [ ] CONFIG_PCI_MSI=y 或 =m
- [ ] CONFIG_PCIEAER=y 或 =m

## 实验 4：QEMU virt PCIe 环境快速验证

**目标**：验证 QEMU virt 机器模型可以正常启动

**步骤**：
```bash
# 创建一个测试磁盘镜像（如果不存在）
qemu-img create -f qcow2 /tmp/virtpcitest.qcow2 4G

# 启动 QEMU virt，等待 3 秒后自动退出（-nographic -monitor none）
timeout 5 /usr/local/bin/qemu-system-x86_64 \
    -M virt \
    -m 256M \
    -display none \
    -nographic \
    -enable-kvm \
    -machine virt \
    2>&1 | head -20 || true

# 如果看到 "pci_host_bridge" 相关日志，说明 PCIe RC 模拟正常
```

**预期结果**：QEMU 正常启动，无崩溃

**验收标准**：
- [ ] QEMU virt 机器可启动
- [ ] 无 KVM 权限错误（除非没有嵌套虚拟化）
- [ ] 进程正常退出（timeout 后）
