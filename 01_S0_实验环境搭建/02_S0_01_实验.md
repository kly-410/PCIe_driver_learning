# S0_01_实验 — 实验环境搭建

> 本节为纯环境验证，共 5 个实验，全部在宿主机执行，不需要进入 QEMU Guest。
>
> **平台说明**：QEMU 9.0.0 编译为 aarch64-softmmu（ARM 64位），使用 `-M virt` 机器模型。
> virt 是 ARM 平台的 PCIe RC 模拟，设备模型比 x86 更干净，是 PCIe 学习推荐平台。

---

## 实验 1：QEMU 版本验证

**目标**：确认 QEMU 9.0.0 aarch64 版本已正确安装

**步骤**：
```bash
### 检查 QEMU 版本
qemu-system-aarch64 --version
### 预期输出：QEMU emulator version 9.0.0

### 或完整路径
/usr/local/bin/qemu-system-aarch64 --version
```

**预期结果**：输出 QEMU emulator version 9.0.0

**验收标准**：
- [ ] `qemu-system-aarch64 --version` 输出包含 "9.0.0"

---

## 实验 2：开发工具链验证

**目标**：确认 GCC / GDB / Linux headers 完整

**步骤**：
```bash
### GCC
gcc --version | head -1

### GDB Python 支持（调试内核模块必需）
gdb -batch -ex "python print('GDB python OK')"

### Make
make --version | head -1

### Linux headers
ls /lib/modules/$(uname -r)/build/Makefile 2>/dev/null && echo "headers OK"
```

**预期结果**：所有工具返回版本号或 OK

**验收标准**：
- [ ] GCC 版本 ≥ 13
- [ ] GDB Python 扩展可用（输出 GDB python OK）
- [ ] Make 可用
- [ ] linux-headers Makefile 存在（输出 headers OK）

## 实验 3：PCI 子系统配置验证

**目标**：确认内核 PCI 配置完整

**步骤**：
```bash
### 检查关键 PCI 配置选项
grep -E "CONFIG_PCI=|CONFIG_PCI_MSI=|CONFIG_PCIEAER=|CONFIG_PCIEPORTBUS=" \
    /boot/config-$(uname -r)
```

**预期结果**：所有关键配置为 y 或 m，无 =n

**验收标准**：
- [ ] CONFIG_PCI=y 或 =m
- [ ] CONFIG_PCI_MSI=y 或 =m
- [ ] CONFIG_PCIEAER=y 或 =m

---

## 实验 4：QEMU virt PCIe 环境快速验证

**目标**：验证 QEMU virt 机器模型可以正常启动

> 注意：`virt` 是 ARM (aarch64) 的机器类型，必须用 `qemu-system-aarch64`，不能用 x86_64 版本。

**步骤**：
```bash
### 创建测试磁盘镜像
qemu-img create -f qcow2 /tmp/virtpcitest.qcow2 4G

### 启动 QEMU virt，等待 5 秒后自动退出
timeout 5 qemu-system-aarch64 \
    -M virt \
    -m 256M \
    -display none \
    -nographic \
    2>&1 | head -20 || true

### 注意：aarch64 + virt 模式下 -enable-kvm 不支持（桌面 CPU 无 ARM 虚拟化扩展）
### 如果看到 virt 或 pci 相关信息，说明 PCIe RC 模拟正常
```

**预期结果**：QEMU 正常启动，无崩溃

**验收标准**：
- [ ] QEMU virt 机器可启动
- [ ] 无 KVM 权限错误
- [ ] 进程正常退出（timeout 后）

## 附：QEMU 编译参考（遇到问题时的参考）

如果需要重新编译 QEMU，执行以下步骤：

```bash
#### 1. 进入源码目录
cd /opt/qemu-9.0.0

#### 2. 配置（选择 aarch64 virt 平台）
sudo ./configure --target-list=aarch64-softmmu --enable-kvm

#### 3. 编译（多核并行，约 20-30 分钟）
sudo make -j$(nproc)

#### 4. 安装
sudo make install

#### 5. 验证（安装路径是 /usr/local/bin/）
qemu-system-aarch64 --version
/usr/local/bin/qemu-system-aarch64 --version
```

关键 configure 选项说明：
- `--target-list=aarch64-softmmu`：只编译 ARM 64 位纯软件模拟模式（virt 机器类型需要）
- `--enable-kvm`：启用硬件加速（需要 CPU 支持 Intel VT-x 或 AMD-V）
