# S1_03_实验 — PCI/PCIe 配置空间（ECAM）

---

## 实验 1：ECAM 地址计算与配置空间读取

**目标**：用 `setpci` 手工计算 ECAM 地址，验证配置空间可访问。

**前置**：ECAM 基址可以通过 `dmesg | grep MCFG` 查找。

**步骤**：
```bash
### 查看 ECAM 基址
sudo cat /proc/ioports | head -5
lspci -nn

### 计算 00:00.0（RC）的 Vendor ID 地址
### ECAM = 0xef800000（假设），Bus=0, Dev=0, Func=0, Offset=0x00
### = 0xef800000 + (0<<20) + (0<<15) + (0<<12) + 0x00
python3 -c "print('ECAM Bus0Dev0Func0:', hex(0xef800000 + (0<<20) + (0<<15) + (0<<12)))"

### 用 setpci 读 RC 的 Vendor ID
sudo setpci -s 00:00.0 0.w
### 预期：Intel 或硬件平台对应的 Vendor ID（如 8086=Intel）

### 读 EP（01:00.0）的 Vendor ID
sudo setpci -s 01:00.0 0.w
```

**验收标准**：
- [ ] 能手工计算 ECAM 地址
- [ ] setpci 读取结果和 lspci -nn 中的 Vendor ID 一致

## 实验 2：Capability 链表遍历

**目标**：用 setpci 手工遍历 Capability 链表，找到 MSI Cap。

**步骤**：
```bash
### 找第一个 Capability 指针（Header Type + 1，通常是 0x34）
sudo setpci -s 01:00.0 34.b   # Capability Pointer

### 读第一个 Capability ID
sudo setpci -s 01:00.0 35.b  # Capability ID（如果 Cap Pointer=0x34）

### MSI Cap ID = 0x05，Power Cap ID = 0x01
#### 从 0x34 开始逐个找，直到 ID = 0xFF（链表结束）
```

**手工遍历示例**：
```
0x34: 0x60  → Capability ID = 0x60? 不是，继续
0x60: 0x05  → MSI Cap (ID=0x05)! 找到了
0x61: 0x70  → Next = 0x70
0x70: 0x01  → Power Cap ID = 0x01
0x71: 0xFF  → 结束
```

**验收标准**：
- [ ] 能手工走完 Capability 链表
- [ ] 能识别 MSI Cap（ID=0x05）和 PCIe Cap（ID=0x10）

## 实验 3：BAR 解析

**目标**：读 BAR 值，理解 BAR 位定义。

**步骤**：
```bash
### 读 EP（01:00.0）的 BAR0
sudo setpci -s 01:00.0 10.l

### BAR 值解读（64-bit 系统）：
### Bit[0] = 0 → Memory Space
### Bit[2:1] = 00 → 32-bit BAR
### Bit[3] = 预取
### Bit[31:4] = 基址

### 对比 lspci 输出
sudo lspci -nn -s 01:00.0 | grep "Memory"
```

**验收标准**：
- [ ] 能解读 BAR 值每一位含义
- [ ] lspci 输出和 setpci 读取结果一致

## 附：配置空间速查

| 偏移 | 内容 | 说明 |
|------|------|------|
| 0x00 | Vendor ID | 厂商 ID（0x8086=Intel）|
| 0x02 | Device ID | 设备 ID |
| 0x06-0x07 | Status / Command | 状态/控制 |
| 0x0E | Header Type | 0x00=EP, 0x01=Bridge |
| 0x10-0x27 | BAR[0-5] | 基地址寄存器 |
| 0x34 | Capability Pointer | 第一个 Capability 偏移 |
| 0x3C | Interrupt Pin | INTx 中断引脚 |
