## AI 评估问题

**Q1. ECAM 基址是 0xef800000，Bus=3, Dev=10, Func=7，访问 Device ID（偏移=0x02）的物理地址是多少？**

预期：用公式计算 = 0xef800000 + (3<<20) + (10<<15) + (7<<12) + 0x02

---

**Q2. Type 0 和 Type 1 配置头部的本质区别是什么？哪个设备用哪个？**

预期：Type 0（EP/终端设备），Type 1（RC/Bridge/Switch 上游端口）；Type 1 比 Type 0 多了 Bus 号范围、I/O 和 Memory 地址范围字段。

---

**Q3. 如果 BAR 值读出来是 0xFFFFFFFF，说明什么？**

预期：BAR 未被分配地址，驱动或 BIOS 还没有往 BAR 写入有效基址。通常意味着设备没有被正确使能，或者驱动还没完成 probe。

---

**Q4. 64-bit BAR 和 32-bit BAR 的本质区别是什么？为什么 BAR[2] 和 BAR[3] 不能凑成一对？**

预期：64-bit BAR 实际占用两个相邻 BAR 寄存器（n 和 n+1），共同表示一个 64-bit 地址；BAR[3] 和 BAR[4] 之间没有硬性禁止，但规范要求一对 BAR 必须是相邻的（n 和 n+1），且如果用 BAR[n] 的 bit[2:1]=10 标记了 64-bit，BAR[n+1] 就不能独立作为 32-bit 使用。

---

**Q5. 为什么 Capability 链表不能无限遍历？链表结束的标志是什么？**

预期：链表以 0xFF（ID=0xFF）结尾；或者 Next Pointer = 0 表示结束；没有结束标志会死循环。
