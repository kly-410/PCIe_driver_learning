# S1_03_实验 — 配置空间与 ECAM

> 前提：安装 libpci-dev
> `sudo apt install libpci-dev`
>
> ECAM 直接访问需要 root 权限。

---

## 实验 1：ECAM 直接访问配置空间

**目标**：通过 /dev/mem 直接读取 ECAM 内存映射，不依赖 libpci。

**步骤**：
```bash
# 编译
make ecam_access

# 读取 RC 的 Vendor/Device ID
sudo ./S1_03_ecam_access 00:00.0 00

# 读取 RC 的 Command/Status
sudo ./S1_03_ecam_access 00:00.0 04

# 读取 RC 的 Class Code / Revision
sudo ./S1_03_ecam_access 00:00.0 08

# 读取 RC 的 Header Type
sudo ./S1_03_ecam_access 00:00.0 0e

# 读取 RC 的 BAR0（如果 RC 有 BAR）
sudo ./S1_03_ecam_access 00:00.0 10
```

**预期结果**：
- 00:00.0 VID=0x1b36（QEMU virt RC），DID=0x0008

**验收标准**：
- [ ] 能编译并运行 ecam_access
- [ ] VID/DID 读取值和 lspci -nn 一致
- [ ] 能解释 ECAM 地址的计算方式

## 实验 2：Capability 链表遍历

**目标**：遍历 PCIe 设备的 Capability 链表，识别所有 Cap。

**步骤**：
```bash
# 编译
make cap_traverse

# 遍历 RC 的 Capability
sudo ./S1_03_cap_traverse 00:00.0

# 遍历 EP 的 Capability（virtio-net 或 nvme）
sudo ./S1_03_cap_traverse 01:00.0

# 对比两者（RC 有 PCIe Cap，virtio-net 设备可能有 MSI + PCIe）
```

**预期结果**：00:00.0（RC）有 PCIe Cap。EP 通常有 PCIe Cap + MSI Cap。

**验收标准**：
- [ ] 能看到完整的 Capability 链表
- [ ] 理解 PCIe Cap ID = 0x10，MSI Cap ID = 0x05
- [ ] 能说出"链表末尾 next_ptr = 0x00"

## 实验 3：Type 0 vs Type 1 Header 识别

**目标**：读取 Header Type 寄存器，识别设备类型。

**步骤**：
```bash
# 读取 RC 的 Header Type（偏移 0x0E，byte）
sudo ./S1_03_ecam_access 00:00.0 0e
# 预期：Type 1（RC = bridge）

# 读取 EP 的 Header Type
sudo ./S1_03_ecam_access 01:00.0 0e
# 预期：Type 0（EP = endpoint）

# 用 setpci 交叉验证
sudo setpci -s 00:00.0 0e.b   # RC Header Type
sudo setpci -s 01:00.0 0e.b   # EP Header Type
```

**验收标准**：
- [ ] 00:00.0 Header Type = 0x01（Type 1，RC/Bridge）
- [ ] 01:00.0 Header Type = 0x00（Type 0，EP）
- [ ] 能解释 Type 0 和 Type 1 的本质区别

## 实验 4：关键 Capability 深度解析

**目标**：解析 MSI Cap 和 PCIe Cap 的关键字段。

**步骤**：
```bash
# 读取 MSI Cap 的 Message Control 寄存器
# MSI Cap 在 Capability 链表中的偏移需要先找到
# PCIe Cap 通常在 offset 0x70 左右（QEMU virt）

# 读 PCIe Cap Device Type（通过 libpci 更简单）
sudo ./S1_03_cap_traverse 00:00.0 | grep -i "Device Type\|Max_Payload"
# 预期：Root Port

# 对比 MPS（Max_Payload_Size）限制
sudo ./S1_03_cap_traverse 00:00.0
sudo ./S1_03_cap_traverse 01:00.0

# 用 setpci 读 PCIe Cap + Link Status
sudo setpci -s 00:00.0 6c.w   # PCIe Cap + Device Control
sudo setpci -s 00:00.0 72.w   # PCIe Cap + Link Status
```

**验收标准**：
- [ ] 能用 cap_traverse 找到 PCIe Cap
- [ ] 能读出 Max_Payload_Size
- [ ] 能用 setpci 交叉验证
