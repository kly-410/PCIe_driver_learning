# S1_02_实验 — 链路层与 Flow Control + DOE

> 前提：安装 libpci-dev
> `sudo apt install libpci-dev`

---

## 实验 1：DOE Mailbox 扫描

**目标**：扫描系统中所有 PCIe 设备的 DOE Mailbox，记录哪些设备支持 DOE。

**步骤**：
```bash
# 编译
make doe

# 扫描 RC（通常没有 DOE）
sudo ./S1_02_doe 00:00.0

# 扫描 PCIe Switch upstream 端口（如果有）
sudo ./S1_02_doe 01:00.0

# 遍历所有 PCIe 设备
for dev in $(lspci -nn | grep "PCIe" | awk '{print $1}'); do
    echo "=== $dev ===" && sudo ./S1_02_doe $dev 2>/dev/null || true
done
```

**预期结果**：大多数消费级设备没有 DOE Mailbox（正常）。服务器级 GPU / NVMe 可能包含 DOE/VPD。

**验收标准**：
- [ ] 编译无错误，生成 `S1_02_doe` 可执行文件
- [ ] `sudo ./S1_02_doe 00:00.0` 输出 "无 DOE Mailbox"
- [ ] 了解 DOE Cap ID = 0x23

## 实验 2：PCIe 链路状态读取

**目标**：读取链路协商速率、宽度、LTSSM 状态，对比 RC 和 EP。

**步骤**：
```bash
# 编译
make link_status

# 读取 RC 链路状态
echo "=== RC (00:00.0) ==="
sudo ./S1_02_link_status 00:00.0

# 读取 EP 链路状态（virtio-net 或 nvme）
echo "=== EP (01:00.0) ==="
sudo ./S1_02_link_status 01:00.0 2>/dev/null || echo "01:00.0 不存在"

# 读取所有设备的链路状态
for dev in $(lspci -nn | grep "PCIe" | awk '{print $1}'); do
    sudo ./S1_02_link_status $dev 2>/dev/null || true
done
```

**预期结果**：RC 通常 Gen3/x16，EP 协商到 Gen3/x1（QEMU virt 限制）。

**验收标准**：
- [ ] RC 协商速率 ≥ Gen2
- [ ] 能区分 L0（训练完成）vs Recovery/Polling（训练中）
- [ ] 对比 RC 和 EP 的协商宽度，理解 x16 RC 可以带 x1 EP

## 实验 3：Flow Control 相关寄存器读取

**目标**：读取 Max\_Payload\_Size 和 Max\_Read\_Request\_Size，理解 FC 消耗速度。

**步骤**：
```bash
# 编译
make fc

# 对比 RC 和 EP 的 MPS / MRRS
echo "=== RC (00:00.0) ==="
sudo ./S1_02_fc 00:00.0

echo "=== EP (01:00.0) ==="
sudo ./S1_02_fc 01:00.0 2>/dev/null || echo "无 EP"

# 用 setpci 交叉验证（PCIe Cap + Device Control offset）
sudo setpci -s 00:00.0 68.w   # DC (Device Control) on RC
sudo setpci -s 01:00.0 68.w   # DC on EP
```

**预期结果**：EP 的 Max\_Payload\_Size 通常 ≤ RC 的 Max\_Payload\_Size（取较小值作为链路限制）。

**验收标准**：
- [ ] 能读出 MPS 和 MRRS 的值
- [ ] 能解释为什么 MPS 大会导致 FC Credit 消耗更快
- [ ] 理解 FC 超额时链路的行为

## 实验 4：链路训练失败模拟（可选）

**目标**：通过拔掉虚拟网线模拟链路断开，观察 LTSSM 变化。

**步骤**：
```bash
# 在 QEMU 中启动 virtio-net
timeout 30 /opt/qemu/bin/qemu-system-x86_64 \
    -M virt -m 256M -nographic \
    -device virtio-net-pci,addr=01.0 \
    2>&1 | grep -i "link\|ltssm\|pcie" &
sleep 5
kill %1 2>/dev/null
echo "链路断开实验：QEMU 窗口关闭后，dmesg 检查链路状态"
dmesg | grep -i "pcie\|link" | tail -10
```

**预期结果**：链路断开后系统日志显示链路进入 Detect 或 Recovery。

**验收标准**：
- [ ] 能观察链路状态变化
- [ ] 能描述 LTSSM 状态转换路径（正常：Detect→Polling→Config→L0）
