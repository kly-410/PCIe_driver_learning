### 改错题（Challenge）

代码中故意埋了一个 bug，运行或审查后找出问题。

**实验代码**（`S2_04_msi.c`，在 `09_S2_04_MSI中断路由/` 目录下）：

```c
/* S2_04_msi.c —— 故意埋了 MSI 门铃地址错误 bug */
#include <linux/pci.h>
#include <linux/msi.h>
#include <linux/module.h>

static void msi_handler(int vector, void *data)
{
    printk(KERN_INFO "MSI interrupt %d received\n", vector);
}

static int my_probe(struct pci_dev *dev, const struct pci_device_id *id)
{
    int ret;
    struct msi_desc *desc;

    /* 读取 MSI Capability 的 Message Address 和 Message Data */
    u16 msg_addr_lo = pci_find_capability(dev, PCI_CAP_ID_MSI);
    u16 msg_addr_offset = /* ... 解析 MSI Cap 拿到 Address Low ... */
    u32 msg_addr = 0xfee00000;   /* 从 MSI Cap 读出的门铃地址 */
    u16 msg_data = 0x1234;       /* 从 MSI Cap 读出的 Vector Data */

    /* MSI 门铃地址 = 0xfee00000（从 MSI Cap 读出）*/
    /* 但代码把高 32 位写错了，写成了 0xfee00000 而不是 0 */
    u64 doorbell_addr = ((u64)0xfee00000 << 32) | 0xfee00000;  /* 错！*/

    /* 或者这样错：门铃地址用了错误的偏移量 */
    /* MSI Spec 说门铃地址是 64-bit，低 32 位有效，高 32 位 = 0 */
    doorbell_addr = 0xfee00000LL;  /* 正确：高 32 位 = 0，低 32 位 = 0xfee00000 */

    /* 假设我们要手动触发 MSI 写入门铃地址（绕过内核 MSI 框架）*/
    /* 但这里把 Message Data 当成了地址的一部分：*/
    doorbell_addr = 0xfee00000LL + (msg_data << 16);  /* 大错！msg_data 是 Data，不是地址 */

    printk(KERN_INFO "Writing to MSI doorbell 0x%llx\n", doorbell_addr);
    /* 这里本应 iowrite32(msg_data, doorbell_addr) 来触发 MSI */
    /* 但 doorbell_addr 算错了，访问到非法地址 */
    return 0;
}

static struct pci_driver my_driver = {
    .name = "s2_04_msi",
    .probe = my_probe,
};

module_pci_driver(my_driver);
MODULE_LICENSE("GPL");
```

**问题**：MSI 门铃地址计算错误。MSI 的 Message Address 是 64-bit，但**高 32 位必须为 0**（MSI Spec Section 6.2.4），低 32 位是要写入的地址（通常是 0xfee00000）。而 Message Data 是要写入的数据（Vector），不是地址的一部分。

1. 找到门铃地址计算错误的那一行
2. 说明为什么 `msg_data << 16` 不能加到门铃地址里

> 提示：MSI Message Address 来自 MSI Capability 的 "Message Address" 字段（低 32 位），高 32 位固定为 0。Message Data 是另一个字段，写入地址后 CPU 收到中断。

预期答案：门铃地址应为 `0xfee00000LL`（高 32 位 = 0，低 32 位 = 0xfee00000）。错误代码 `doorbell_addr = 0xfee00000LL + (msg_data << 16)` 把 Message Data 混淆为地址偏移量了。正确写法：`u64 doorbell = (u64)msg_addr_low;`（msg_addr_low 来自 MSI Cap 的 Message Address 字段）。
