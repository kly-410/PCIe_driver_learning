# S1_03 — 配置空间与 ECAM

## 本节目标

- 理解 ECAM（Enhanced Configuration Access Mechanism）原理
- 掌握 Type 0 Header（EP）与 Type 1 Header（RC/bridge）的区别
- 掌握 Capability 链表结构与遍历方法
- 掌握关键 Capability：PMCAP、MSICAP、PCIeCAP

## 前置依赖

S1_01（TLP 类型）。

---

## A. 原理（是什么）

### ECAM 原理

PCI 用 IO 端口访问配置空间（0xcf8/0xcfc），但 x86 IO 空间只有 64 bytes，不够用。

ECAM 将配置空间映射到内存地址空间，CPU 直接解引用指针访问：

```
ECAM 地址格式：
[31:20] Bus Number (8 bits)
[19:15] Device Number (5 bits)
[14:12] Function Number (3 bits)
[11:02] Register Number (byte offset / 4)
[01:00] Byte Enable (always 00b for 32-bit access)

ECAM 基址（QEMU virt）：0xef800000（256MB 窗口）
```

QEMU virt 示例：访问 Bus=1, Dev=0, Fn=0, Reg=0
→ ECAM 地址 = 0xef800000 + (1<<20) + (0<<12) + 0 = **0xef900000**

### Type 0 vs Type 1 Header

| 字段 | Type 0（EP）| Type 1（RC/Bridge/Switch Port）|
|------|------------|------------------------------|
| Header Type | 0x00 | 0x01 |
| BAR 寄存器 | 0x010~0x027（6个BAR）| 无 BAR |
| Primary/Secondary Bus | 无 | 有（路由用）|
| Subordinate Bus | 无 | 有 |
| 用途 | 终端设备 | 路由/桥接 |

**判断**：读 Offset 0x0E，bit 7=1 表示多功能，lower 7 bits=0 是 Type 0，=1 是 Type 1。

### Capability 链表

每个 Capability 至少 4 bytes：ID + Next Pointer + Cap-specific registers。

常用 Capability ID：
- 0x01：PM（Power Management）
- 0x05：MSI（Message Signaled Interrupt）
- 0x10：PCIe
- 0x11：MSI-X
- 0x22：DOE（PCIe 6.x）
- 0x23：DOE Mailbox

遍历：从 Offset 0x34（Type 0）或 0x38（Type 1）的 Capabilities Pointer 开始，顺着 next_ptr 遍历直到 next_ptr=0x00。

## B. 为什么这样设计

### 为什么用内存映射而非 IO 端口？

x86 IO 空间仅 64 bytes（port 0xcf8~0xcff），PCIe 支持 256 Bus × 32 Dev × 8 Func × 4KB = 8GB 配置空间，IO 端口根本不够。ECAM 通过内存地址直接访问，地址空间几乎是无限的。

### 为什么 Capability 用链表而非数组？

不同设备支持的功能集差异巨大（有的有 MSI，有的有 SR-IOV，有的有 AER）。链表灵活性高，设备可以按需声明 Capability，新增 Cap 不影响旧有设计。

### MSI 为什么比 INTx 好？

INTx 使用边带中断引脚（INTA/B/C/D），需要 CPU 引脚 + 共享 + 引脚数量限制（最大 4）。MSI 是内存写事务，直接写到 RC 分配的 MSI 地址，CPU 中断控制器接收，无引脚共享，支持更多向量（最多 32）。

## C. 常见错误案例

### 错误 1：Capability 链表断裂（读出 0xFF）

**现象**：`pci_find_capability()` 返回 0，配置空间某处出现 0xFF。

**排查**：确认 Vendor/Device ID 正确（0xFFFF 表示无设备），检查 Bus/Dev/Func 是否匹配。

### 错误 2：ECAM 地址越界

**现象**：MMIO 访问配置空间返回全 F，触发 Page Fault。

**排查**：确认 MCFG 表的 ECAM 基址，`dmesg | grep MCFG` 或 `cat /sys/firmware/acpi/tables/MCFG`。

## D. 决策树

```
需要访问 PCIe 配置空间
├── 目的：读设备 VID/DID
│   └── TLP 类型：CfgRd → ECAM 地址 = 基址 + (bus<<20) + (devfn<<12) + 0
├── 目的：读 BAR 地址
│   └── TLP 类型：CfgRd → BAR 寄存器偏移（0x010~0x027）
├── 目的：配置 MSI 中断
│   └── 找 MSI Cap（Cap ID=0x05）→ 读/写 MSICAP_MSK / MD
└── 目的：读取链路速率/宽度
    └── 找 PCIe Cap（Cap ID=0x10）→ Link Status Register
```
