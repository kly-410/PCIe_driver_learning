## AI 评估问题

**Q1. ECAM 地址 = 基址 + (bus<<20) + (devfn<<12) + offset。如果基址=0xef800000，bus=2, dev=3, fn=1, offset=0，计算物理地址。**

预期：devfn = (3<<3)|1 = 25 = 0x19。地址 = 0xef800000 + (2<<20) + (0x19<<12) + 0 = 0xef800000 + 0x0020_0000 + 0x19_000 = **0xefa9_0000**。

---

**Q2. Type 0 Header 和 Type 1 Header 的本质区别是什么？**

预期：Type 0（EP）有 6 个 BAR 寄存器，不能转发 TLP。Type 1（RC/Bridge）有 Primary/Secondary/Subordinate Bus Number 字段，用于在 Switch/Bridge 中路由 TLP，不能直接用于 EP。EP 可以通过软件配置变成 RC——这是固件/硬件拓扑决定的，不是 PCI 协议层面的转换。

---

**Q3. Capability 链表的遍历终止条件是什么？读到 next_ptr = 0xFF 意味着什么？**

预期：next_ptr = 0x00 表示链表结束，正常终止。读到 next_ptr = 0xFF 表示 Capability 结构损坏（PCI Spec 规定 next_ptr 有效值是 4-byte aligned，不可能是 0xFF），应停止遍历。

---

**Q4. pci_read_config_word() 最终发出的是什么 TLP？它是 Posted 还是 Non-Posted？**

预期：CfgRd（Configuration Read），Non-Posted。RC 发 CfgRd 到目标设备的配置空间，EP 必须返回 CplD 携带数据。如果 CfgRd 是 Posted，RC 无法知道配置寄存器值，读取操作无法完成。

---

**Q5. MSI Cap 的 Message Address 字段的含义是什么？为什么它是 RC 侧的地址而非 EP 的 BAR？**

预期：MSI Address 是 RC 分配给 EP 的一个 MMIO 地址（不是 EP 的 BAR，是 RC 侧 MSI 中断控制器的邮箱地址）。EP 驱动写这个地址（携带向量号），RC 侧的 MSI 中断控制器收到后产生 CPU 中断。这本质是一个门铃机制——地址是"门铃编号"，数据是"按下的号码"。
