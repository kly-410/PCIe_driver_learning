## A. 原理

### INTx vs MSI

**INTx（传统 PCIe 中断）**：
- INTx 是带内消息（Assert_INTx / Deassert_INTx）
- 需要单独的中断引脚（INTA/B/C/D）
- 所有设备共享一个 CPU 中断线，依赖中断控制器（PIC/APIC）
- 缺点：需要 CPU 介入，效率低

**MSI（Message Signaled Interrupt）**：
- MSI 是用 **MWr TLP** 携带中断消息，发送到 RC 的 MSI 地址
- 不需要中断引脚，直接写到内存地址
- 每个向量独占一个地址，支持多向量

### MSI-X：MSI 的升级版

| 特性 | MSI | MSI-X |
|------|-----|-------|
| 向量数 | 1-32 | 1-2048 |
| 表位置 | 配置空间内 | BAR映射外置，2048个条目 |
| 中断迁移 | 困难（需修改 MSI Cap）| 容易（修改BAR映射表即可）|
| 使用场景 | 简单设备 | 网卡、NVMe 等多向量设备 |

### MSI 启用流程

源码：`drivers/pci/msi/msi.c:423`

```c
int __pci_enable_msi_range(struct pci_dev *dev, int minvec, int maxvec,
                             struct irq_affinity *affd)
{
    int nvec;

    /* 1. 检查 MSI 是否支持（pci_msi_supported）*/
    if (!pci_msi_supported(dev, minvec) || dev->current_state != PCI_D0)
        return -EINVAL;

    /* 2. 不能同时启用 MSI-X */
    if (dev->msix_enabled) {
        pci_info(dev, "can't enable MSI (MSI-X already enabled)\n");
        return -EINVAL;

    /* 3. 获取设备能发送的 MSI 向量数（读 MSI Cap）*/
    nvec = pci_msi_vec_count(dev);          /* drivers/pci/msi/msi.c:494 */
    if (nvec < 0)
        return nvec;
    if (nvec < minvec)
        return -ENOSPC;

    /* 4. 建立 MSI 上下文（分配 Linux IRQ 号）*/
    rc = pci_setup_msi_context(dev);
    if (rc)
        return rc;

    /* 5. 建立 MSI 中断域（irq_domain）*/
    if (!pci_setup_msi_device_domain(dev, nvec))
        return -ENODEV;

    /* 6. 分配实际向量数（可能比请求的少）*/
    if (nvec > maxvec)
        nvec = maxvec;

    /* 7. 向量亲和性处理（多核负载均衡）*/
    for (;;) {
        if (affd) {
            nvec = irq_calc_affinity_vectors(minvec, nvec, affd);
            if (nvec < minvec)
                return -ENOSPC;
        }
        // ...
    }
}
```

### `struct msi_desc` — MSI 中断描述符

源码：`include/linux/msi.h:183`

```c
struct msi_desc {
    unsigned int        irq;           // Linux IRQ 号
    unsigned int        nvec_used;      // 本条目使用的向量数
    struct device      *dev;            // 所属设备
    struct msi_msg      msg;           // MSI 消息（地址+数据）
    struct irq_affinity_desc *affinity; // CPU 亲和性掩码

    void (*write_msi_msg)(struct msi_desc *entry, void *data);
    void *write_msi_msg_data;

    u16                  msi_index;    // MSI-X 中是索引，MSI 中未用
    union {
        struct pci_msi_desc  pci;      // PCI MSI 特有字段
        struct msi_desc_data data;    // 平台私有数据
    };
};
```

### 驱动使用 MSI 的典型代码

```c
/* 在驱动 probe 里 */
int nvec = 4;  /* 申请 4 个中断向量 */

ret = pci_enable_msi_block(dev, nvec);
if (ret) {
    dev_warn(&dev->dev, "MSI enable failed: %d, falling back to INTx\n", ret);
    pci_intx(dev, 1);  /* 回退到 INTx */
    return 0;
}

/* 成功：dev->irq 是第一个 IRQ 号（其余是 +1, +2, +3）*/
for (i = 0; i < nvec; i++) {
    ret = request_irq(dev->irq + i, handler, 0, DRIVER_NAME, dev);
    if (ret) {
        dev_err(&dev->dev, "request_irq %d failed\n", dev->irq + i);
        /* 清理已申请的中断 */
        while (i-- > 0)
            free_irq(dev->irq + i, dev);
        pci_disable_msi(dev);
        return ret;
    }
}
```

## B. 决策树

```
设备需要中断
  ├── 单向量，设备简单 → MSI（pci_enable_msi_block）
  └── 多向量高性能（网卡/NVMe）→ MSI-X（pci_enable_msix）

MSI 使能失败
  → 回退到 INTx（pci_intx(dev, 1)）
  → 记录警告但不让驱动加载失败

设备卸载
  → free_irq(irq, dev) × nvec
  → pci_disable_msi(dev)
```
