## A. 原理

### INTx vs MSI

**INTx（传统 PCIe 中断）**：
- INTx 是带内消息（Assert_INTx / Deassert_INTx）
- 需要单独的中断引脚（INTA/B/C/D）
- 所有设备共享一个 CPU 中断线，依赖中断控制器（PIC/APIC）
- 缺点：需要 CPU 介入，效率低

**MSI（Message Signaled Interrupt）**：
- MSI 是用 **MWr TLP** 携带中断消息，发送到 RC 的 MSI 地址
- 不需要中断引脚，直接写到内存地址
- 每个向量独占一个地址，支持多向量

### MSI-X：MSI 的升级版

| 特性 | MSI | MSI-X |
|------|-----|-------|
| 向量数 | 1-32 | 1-2048 |
| 表位置 | 配置空间内 | BAR映射外置，2048个条目 |
| 中断迁移 | 困难（需要修改 MSI Cap）| 容易（修改BAR映射表即可）|
| 使用场景 | 简单设备 | 网卡、NVMe 等多向量设备 |

### MSI 启用流程

```c
/* 在驱动 probe 里 */
int nvec = 4;  /* 申请 4 个中断向量 */

int ret = pci_enable_msi_block(dev, nvec);
if (ret) {
    /* 失败，回退到 INTx 或报错 */
    dev_warn(&dev->dev, "MSI enable failed: %d\n", ret);
    return ret;
}

/* 成功分配了 nvec 个向量，Linux 中分配了 Linux IRQ 号
 * 通过 dev->irq 获得第一个 IRQ（其余是 dev->irq + 1, + 2, + 3）
 */
request_irq(dev->irq, handler, 0, DRIVER_NAME, dev);
```

## B. 决策树

```
设备需要中断
  ├── 简单设备，单向量 → MSI（pci_enable_msi）
  └── 多向量高性能设备（网卡/NVMe）→ MSI-X（pci_enable_msix）

MSI 使能失败
  → 回退到 INTx（pci_intx(dev, true)）
  → 记录错误但不让驱动加载失败
```
