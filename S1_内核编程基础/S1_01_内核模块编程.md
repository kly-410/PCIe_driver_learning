# S1_01_内核模块编程

## 一、内核模块是什么

内核模块是 Linux 内核的"插件"——在内核已经运行的情况下，可以动态加载一段代码到内核空间执行，卸载时再移除。驱动是最常见的内核模块。

PCIe 主机驱动在真实硬件上通常是**内置**的（因为 RC 必须先存在才能枚举设备），EP 驱动通常是**模块**。

## 二、最小模块框架

```c
#include <linux/init.h>
#include <linux/module.h>
#include <linux/kernel.h>

MODULE_LICENSE("GPL");
MODULE_AUTHOR("kly");
MODULE_DESCRIPTION("PCIe Driver Learning - Step 1");
MODULE_VERSION("1.0");

static int __init skeleton_init(void)
{
    printk(KERN_INFO "skeleton: loaded\n");
    return 0;  // 0 = success，负数 = 错误码如 -ENOMEM
}

static void __exit skeleton_exit(void)
{
    printk(KERN_INFO "skeleton: unloaded\n");
}

module_init(skeleton_init);
module_exit(skeleton_exit);
```

逐行解析：

| 行 | 作用 |
|----|------|
| `MODULE_LICENSE("GPL")` | 许可证声明，GPL 才能用大多数内核符号 |
| `__init` 修饰 | 函数放入 .init 段，初始化后释放内存 |
| `return 0` | 成功返回 0；失败返回负数错误码 |
| `module_init()` | 注册加载时调用的函数 |
| `module_exit()` | 注册卸载时调用的函数 |

## 三、MODULE_LICENSE

| 许可证 | 符号访问 | 工程建议 |
|-------|---------|---------|
| `GPL` | 全部 | ✅ 驱动都用这个 |
| `GPL v2` | 全部 | 限定GPLv2 |
| `Dual BSD/GPL` | 全部 | BSD或GPL任选 |
| `Proprietary` | 仅 `__this_module` | ❌ 大量符号用不了，不要用 |

**生产驱动必须用 `GPL`**，否则 `container_of`、`mutex_lock` 等核心宏都用不了。

## 四、__init 和 __exit 本质

```c
// 内核源码定义
#define __init    __section(".init.text")
#define __exit    __section(".exit.text")
```

`__init` 的作用：
1. 函数放入 `.init.text` ELF段
2. 内核初始化完成后，`.init` 段整体释放（节省内存）
3. **只能从 `module_init` 调用一次**

`__exit` 类似：模块卸载时调用；内置驱动（不是模块）则忽略。

## 五、EXPORT_SYMBOL

```c
// my_driver.c
int my_pcie_init(void) { ... }
EXPORT_SYMBOL(my_pcie_init);  // 导出后其他模块可用
```

引用未导出符号 → 编译时报 `Unknown symbol in module`。

## 六、模块编译链路

```
.c → .o（make）→ .ko（insmod）→ modpost（验证）→ 内核执行
```

`modinfo module.ko` 可查看模块元信息。

## 七、printk 的工程替代

生产代码不用 `printk(KERN_ERR ...)`，用：

```c
dev_err(&dev->dev, "failed: %d\n", ret);   // 有device指针时，包含设备路径
pr_err("pcie: failed\n");                  // 无device时，不含设备路径
```

`KERN_INFO` / `KERN_DEBUG` 同理。

## 八、module_param

```c
static int msi_vectors = 8;
static int debug = 0;
module_param(msi_vectors, int, 0644);
module_param(debug, int, 0644);
MODULE_PARM_DESC(msi_vectors, "Number of MSI vectors");
MODULE_PARM_DESC(debug, "Enable debug (0=off)");
```

加载时传参：`sudo insmod kmod_skeleton.ko debug=1 msi_vectors=4`

查看：`modinfo kmod_skeleton.ko | grep -E "^parm:"`
