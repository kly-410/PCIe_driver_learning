## A. 原理

### `struct resource` — Linux 地址范围抽象

源码：`include/linux/ioport.h:22`

```c
struct resource {
    resource_size_t start;    /* 地址范围起始（物理地址）*/
    resource_size_t end;      /* 地址范围结束（包含，大于等于 start）*/
    const char     *name;     /* 资源名，如 "PCI IO" */
    unsigned long   flags;    /* IORESOURCE_* 标志 */
    unsigned long   desc;     /* 扩展描述符 */
    struct resource *parent, *sibling, *child;  /* 资源树节点指针 */
};
```

**关键 flags**（`include/linux/ioport.h:40`）：

| Flag | 值 | 含义 |
|------|-----|------|
| `IORESOURCE_IO` | 0x100 | I/O 端口地址空间 |
| `IORESOURCE_MEM` | 0x200 | MMIO 物理内存地址空间 |
| `IORESOURCE_PREFETCH` | 0x2000 | 可预取内存（CPU 可以投机性读取）|
| `IORESOURCE_MEM_64` | 0x100000 | 64-bit 内存地址 |

**资源树结构**：`pci_dev.resource[0-5]` 对应 BAR0-BAR5，每个是 `struct resource`。

### BAR 分配流程

源码：`drivers/pci/setup-res.c:298`

```c
static int __pci_assign_resource(struct pci_bus *bus, struct pci_dev *dev,
                                  int resno, resource_size_t size,
                                  resource_size_t align)
{
    struct resource *res = pci_resource_n(dev, resno);  /* 取出 BAR 对应的 resource */
    resource_size_t min;

    /* 系统保留的最小地址（IO: 0x100, MEM: 1MB）*/
    min = (res->flags & IORESOURCE_IO) ? PCIBIOS_MIN_IO : PCIBIOS_MIN_MEM;

    /* 优先尝试精确预取匹配 */
    ret = pci_bus_alloc_resource(bus, res, size, align, min,
                                 IORESOURCE_PREFETCH | IORESOURCE_MEM_64,
                                 pcibios_align_resource, dev);
    if (ret == 0)
        return 0;  /* 成功 */

    /* 回退到 32-bit 预取窗口 */
    if ((res->flags & (IORESOURCE_PREFETCH | IORESOURCE_MEM_64))
         == (IORESOURCE_PREFETCH | IORESOURCE_MEM_64)) {
        ret = pci_bus_alloc_resource(bus, res, size, align, min,
                                    IORESOURCE_PREFETCH,
                                    pcibios_align_resource, dev);
        if (ret == 0)
            return 0;
    }

    /* 最后回退到非预取内存窗口 */
    if (res->flags & (IORESOURCE_PREFETCH | IORESOURCE_MEM_64))
        ret = pci_bus_alloc_resource(bus, res, size, align, min, 0,
                                    pcibios_align_resource, dev);
    return ret;
}
```

### /proc/iomem 读取

```
cat /proc/iomem | grep "PCI"
```

每行格式：`起始地址-结束地址 : 资源名`，对应一个 `struct resource`。

### 驱动资源操作 API

```c
/* 在驱动 probe 里 */
pci_resource_start(dev, bar)    // BAR 起始物理地址（pci_resource_n）
pci_resource_len(dev, bar)      // BAR 地址范围长度
pci_resource_flags(dev, bar)     // IORESOURCE_MEM 等标志

void __iomem *pci_iomap(struct pci_dev *dev, int bar, int len);
// 把 BAR 映射到内核虚拟地址（ioremap）

/* 驱动 remove 里 */
pci_iounmap(dev, bar0_addr);   // 解除 MMIO 映射（必须和 pci_iomap 配对）
pci_release_regions(dev);        // 释放 BAR I/O 范围
```

## B. 决策树

```
probe() 里访问设备 MMIO
  → pci_enable_device(dev)           ← 先使能设备
  → pci_request_regions(dev, name)  ← 申请 BAR 范围
  → pci_iomap(dev, bar, 0)          ← 建立 MMIO 映射
  → ioread32(bar0 + offset)         ← 读写 MMIO 寄存器

remove() 里清理
  → pci_iounmap(dev, bar0)          ← 解除映射
  → pci_release_regions(dev)        ← 释放资源
  → pci_disable_device(dev)         ← 关闭设备
```

## C. 常见错误

| 错误现象 | 原因 | 解决方法 |
|---------|------|---------|
| BAR 映射地址读出来全 F | 设备未被 enable，或 BAR 未分配 | 确认调用了 `pci_enable_device` |
| `pci_iomap` 返回 NULL | BAR 地址为 0（未分配）| 打印 `pci_resource_start` 检查 |
| remove 时 kernel panic | iounmap 的地址和 iomap 不匹配 | 记录 iomap 返回值，remove 时用同一个值 |
