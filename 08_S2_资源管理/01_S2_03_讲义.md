## A. 原理

### `struct resource` — Linux 地址范围抽象

```c
struct resource {
    resource_size_t start;   /* 地址范围起始 */
    resource_size_t end;     /* 地址范围结束（包含）*/
    const char *name;         /* 资源名 */
    unsigned long flags;      /* IORESOURCE_MEM / IORESOURCE_IO */
    struct resource *parent, *sibling, *child;
};
```

`pci_dev.resource[i]` 就是 6 个 BAR 对应的 resource。

### BAR 分配流程

```
pci_enable_device()
  → pci_bar_init(dev)
      → 读 BAR 值
      → 判断是否需要分配（BAR = 0xFFFFFFFF）
  → pci_assign_resource(dev, bar)
      → 从iomem_resource 分配一个地址范围
      → 写回 BAR 寄存器
```

### /proc/iomem

```
cat /proc/iomem
```
显示系统所有 MMIO 地址分配，每行一个 resource。

---

## B. 决策树

```
访问 BAR
  → pci_resource_start(dev, bar) = 起始地址
  → pci_resource_len(dev, bar) = 长度
  → pci_iomap(dev, bar, 0) = ioremap 后的虚拟地址
```
