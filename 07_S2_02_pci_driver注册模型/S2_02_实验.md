# S2_02_实验 — pci_driver 注册模型

---

## 实验 1：查看已注册驱动

```bash
ls /sys/bus/pci/drivers/
```

列出所有 PCI 驱动，找到自己加载的驱动名称。

---

## 实验 2：手动触发驱动重新绑定

```bash
### 查看设备对应的驱动
cat /sys/bus/pci/devices/0000:01:00.0/driver
echo "0000:01:00.0" | sudo tee /sys/bus/pci/devices/0000:01:00.0/driver/unbind
echo "0000:01:00.0" | sudo tee /sys/bus/pci/drivers/igb/bind
```

### 改错题（Challenge）

代码中故意埋了一个 bug，运行或审查后找出问题。

**实验代码**（`S2_02_driver.c`，在 `07_S2_02_pci_driver注册模型/` 目录下）：

```c
/* S2_02_driver.c —— 故意埋了 pci_driver.name 不匹配 bug */
#include <linux/pci.h>
#include <linux/module.h>

static int my_probe(struct pci_dev *dev, const struct pci_device_id *id)
{
    printk(KERN_INFO "pci: probe called\n");
    return 0;
}

static void my_remove(struct pci_dev *dev)
{
    printk(KERN_INFO "pci: remove called\n");
}

static struct pci_driver my_driver = {
    /* ← name 拼写错误，导致驱动无法和设备匹配 */
    .name   = "my_pci_driver",      /* 代码里是这样写的 */
    .id_table = NULL,
    .probe  = my_probe,
    .remove = my_remove,
};

static int __init driver_init(void)
{
    /* 在 /sys/bus/pci/drivers/ 下会显示为 "my_pci_skeleton" */
    return pci_register_driver(&my_driver);
}

static void __exit driver_exit(void)
{
    pci_unregister_driver(&my_driver);
}

module_init(driver_init);
module_exit(driver_exit);
MODULE_LICENSE("GPL");
```

**问题**：驱动注册后，手动 echo 设备 BDF 到 `/sys/bus/pci/drivers/my_pci_skeleton/bind` 无法成功绑定。

1. 找到 `my_driver.name` 和实际要 bind 的驱动名称不匹配的地方
2. 说明为什么 name 不匹配会导致 bind 失败

> 提示：`/sys/bus/pci/drivers/my_pci_skeleton/` 的目录名来自 `pci_driver.name`。如果目录名和代码中 `pci_register_driver` 注册的 name 不一致，系统找不到对应的驱动。

预期答案：驱动代码中的 `.name = "my_pci_driver"` 但 bind 路径用的是 `my_pci_skeleton`，两者不一致。应将 `.name` 改为 `"my_pci_skeleton"` 以匹配 sysfs 中的目录名，或将目录名改为你实际注册的名字。
