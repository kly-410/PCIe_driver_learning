## A. 原理

### Linux 设备驱动模型三层

```
Bus Type（pci_bus_type）← 总线抽象层
    ↓ register_bus()
Device（struct pci_dev）← 总线上发现的设备
    ↓ pci_scan_bus() 发现，创建 pci_dev
Driver（struct pci_driver）← 注册到总线的驱动
    ↓ pci_register_driver() 注册
    → id_table 匹配成功 → probe()
```

### `pci_driver` 注册实现

源码：`drivers/pci/pci-driver.c:1464`

```c
int __pci_register_driver(struct pci_driver *drv, struct module *owner,
                          const char *mod_name)
{
    /* 初始化 driver 结构体 */
    drv->driver.name  = drv->name;
    drv->driver.bus   = &pci_bus_type;   /* 指定总线类型为 PCI */
    drv->driver.owner = owner;
    drv->driver.mod_name = mod_name;
    drv->driver.groups  = drv->groups;
    drv->driver.dev_groups = drv->dev_groups;

    spin_lock_init(&drv->dynids.lock);
    INIT_LIST_HEAD(&drv->dynids.list);

    /* 注册到 driver core（最终调用 bus->drv_probe）*/
    return driver_register(&drv->driver);
}
EXPORT_SYMBOL(__pci_register_driver);
```

**关键**：`drv->driver.bus = &pci_bus_type` — 这一行把驱动绑定到 PCI 总线。

### id_table 匹配规则

```c
static const struct pci_device_id ids[] = {
    { PCI_DEVICE(0x8086, 0x1234) },           // Intel 0x1234 精确匹配
    { PCI_DEVICE(PCI_ANY_ID, 0x5678) },      // 任何厂商，设备 0x5678
    { PCI_CLASS(PCI_CLASS_STORAGE_IDE, 0xFFFF, 0xFFFF) }, // 按类匹配
    { PCI_DEVICE_SUB(PCI_ANY_ID, PCI_ANY_ID, 0x1234, 0x5678) }, // 子系统匹配
    { 0, }  /* 结束标记，必须有 */
};
```

**匹配逻辑**：内核遍历所有 `pci_dev`，对每个设备遍历 `id_table`，调用 `pci_match_id()`，比较 `vendor + device + subvendor + subdevice + class`。

### `probe` 典型实现（pci_driver 里的成员）

源码：`drivers/pci/pci-driver.c` 中无通用实现，每个驱动自己实现。参考框架：

```c
static int my_probe(struct pci_dev *dev, const struct pci_device_id *id)
{
    /* id 是匹配到的那一项，可以用 id->driver_data 传私有数据 */
    struct my_priv *priv;

    if (pci_enable_device(dev))
        return -EIO;

    priv = devm_kzalloc(&dev->dev, sizeof(*priv), GFP_KERNEL);
    if (!priv)
        return -ENOMEM;
    pci_set_drvdata(dev, priv);          /* 存 driver_data */

    /* 用 id->driver_data 判断是哪个型号，做差异化处理 */
    if (id->driver_data == TYPE_A)
        init_type_a(dev);
    else
        init_type_b(dev);

    return 0;
}
```

## B. 决策树

```
insmod my_pci_driver.ko
  → my_init()
  → pci_register_driver(&my_pci_driver)
  → driver_register(&drv->driver)
  → bus_probe_device(dev)  // 对每个已存在的 pci_dev 尝试匹配
  → pci_match_device(drv, dev)  // 对比 id_table
  → probe(dev, id)  // 匹配成功 → 驱动接管设备
```

## C. 常见错误

| 错误现象 | 原因 | 解决方法 |
|---------|------|---------|
| id_table 空，probe 永远不调用 | 只有 `{0,}` 结束标记 | 填入正确的 Vendor/Device ID |
| modprobe 不自动加载 | 缺少 `MODULE_DEVICE_TABLE(pci, ids)` | 添加此宏 |
| 设备已有驱动，再 insmod 报错 | 设备已绑定，不能重复 | `echo 0000:01:00.0 > /sys/bus/pci/devices/.../driver/unbind` |
