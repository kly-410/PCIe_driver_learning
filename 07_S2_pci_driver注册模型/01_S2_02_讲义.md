# S2_02 — pci_driver 注册模型

## 本节目标

- 理解 Linux 驱动模型三层结构：Bus Type → Driver → Device
- 掌握 `pci_driver` 注册流程和 `id_table` 匹配机制
- 理解 `probe` 和 `remove` 的调用时机

## A. 原理

### Linux 设备驱动模型三层

```
Bus Type（总线层）
    ↓ register_bus()
Device（设备）
    ↓ pci_scan_bus() 发现设备，创建 pci_dev
Driver（驱动）
    ↓ pci_register_driver() 注册到总线
```

PCI 驱动注册时，内核将 `pci_driver.id_table` 和已存在的 `pci_dev` 逐一匹配，匹配成功则立即调用 `probe()`。

### id_table 匹配规则

```c
static const struct pci_device_id ids[] = {
    { PCI_DEVICE(0x8086, 0x1234) }, // Intel 0x1234 设备
    { PCI_DEVICE(PCI_ANY_ID, 0x5678) }, // 任何厂商的 0x5678 设备
    { 0, } // 结束标记
};
```

匹配时同时比较 `vendor + device`，都匹配才调用 `probe`。

---

## B. 决策树

```
pci_register_driver()
  → 遍历所有 pci_dev
  → id_table 匹配成功
      → probe(dev, id) 被调用
      → 返回 0 → 驱动接管设备
```
