## AI 评估问题

**Q1. MRd 需要 CplD 返回数据，而 MWr 不需要。为什么？**

预期：MRd 是 Non-Posted 读，RC 不知道 EP BAR 里存了什么，必须等 EP 返回数据。MWr 是 Posted 写，写完即完成，RC 继续执行下一步，不等 EP 确认（牺牲可靠性换取性能）。

---

**Q2. 地址路由和 ID 路由分别用在哪些场景？**

预期：地址路由用于 Memory Read/Write（MRd/MWr），TLP Header 含目标 PCIe 地址，Switch 查地址范围表决定转发方向。ID 路由用于 Configuration Read/Write（CfgRd/CfgWr）和 Completion（Cpl/CplD），TLP Header 含 Bus/Dev/Func，Switch 查 Bus 号范围决定转发方向。

---

**Q3. Switch 如何知道一个 TLP 应该从哪个下游端口转发？**

预期：地址路由时查地址范围表（Base/Limit 寄存器），匹配到哪个下游端口的地址范围就走哪个口。ID 路由时查 Bus Number 注册表（Secondary/Subordinate Bus 号）。隐式路由时，上游来的广播到所有下游，下游来的转发到上游。

---

**Q4. MSI 本质是什么 TLP？它的地址路由到哪里？**

预期：MSI 本质是一个 Posted Memory Write（MsgD），地址是 RC 分配的 MSI 中断控制器邮箱地址，数据字段携带中断向量号。不是路由到 EP BAR，是路由到 RC 侧的门铃地址。

### 补充说明

**EP 发起事务的限制**：
- EP 只能被动响应来自 RC 的事务（Cfg MRd/MWr、Mem MRd/MWr 等）
- EP 可以主动发起的两类事务：① MSI 中断（Posted MemWr）② 电源管理消息（PME）
- 除上述两类外，EP 不能主动发起任何事务——这与 Switch 不同，Switch 可以转发任何 TLP

**MSI 的完整流程**：
1. 枚举阶段：RC 为 EP 分配 MSI 地址，写入 EP 的 MSI Capability 寄存器
2. 运行时：EP 写入该 MSI 地址 → RC 侧 MSI 控制器捕获 → 触发 CPU 中断
3. 数据字段携带中断向量号，用于区分不同中断源

---

**Q5. RC 和 EP 的本质区别是什么？Linux 内核如何表示 RC？**

预期：RC 是 CPU 与 PCIe 总线的接口，是事务的发起者和终结者，不是单纯的设备，是一组硬件功能的集合（Host Bridge + RCiEP + RCeC）。Linux 内核用 `struct pci_host_bridge` 表示 RC，`struct pci_dev` 表示 EP。EP 不能转成 RC——这是硬件拓扑决定的，不是软件配置出来的。
