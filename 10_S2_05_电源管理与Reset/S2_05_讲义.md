## A. 原理

### PCIe D 状态（D0/D1/D2/D3hot/D3cold）

| 状态 | 说明 | 进入方式 |
|------|------|---------|
| D0 | 全功能工作 | — |
| D1 | 轻睡眠 | PM_PME |
| D2 | 深睡眠 | PM_PME |
| D3hot | 断电热（可唤醒）| PME |
| D3cold | 完全断电 | 主电源断开 |

**电源状态转换**：D3hot ↔ D0 需要重新枚举设备（配置空间内容丢失）。

### `pci_set_power_state` 实现

源码：`drivers/pci/pci.c:1130`

```c
void pci_update_current_state(struct pci_dev *dev, pci_power_t state)
{
    /* 优先查询平台固件（ACPI）*/
    if (platform_pci_get_power_state(dev) == PCI_D3cold) {
        dev->current_state = PCI_D3cold;
    } else if (dev->pm_cap) {
        /* 从 PMCSR（Power Management Control/Status Register）读取状态 */
        u16 pmcsr;
        pci_read_config_word(dev, dev->pm_cap + PCI_PM_CTRL, &pmcsr);
        if (PCI_POSSIBLE_ERROR(pmcsr))
            dev->current_state = PCI_D3cold;  /* 读不到 = D3cold */
        else
            dev->current_state = pmcsr & PCI_PM_CTRL_STATE_MASK;  /* D0/D1/D2/D3 */
    } else {
        dev->current_state = state;
    }
}

/* 平台电源转换（通过 ACPI / platform specific）*/
int pci_platform_power_transition(struct pci_dev *dev, pci_power_t state)
{
    int error;
    error = platform_pci_set_power_state(dev, state);  /* ACPI pm1_cnt */
    if (!error)
        pci_update_current_state(dev, state);
    else if (!dev->pm_cap)
        dev->current_state = PCI_D0;  /* 无 PM capability 就回退到 D0 */
    return error;
}
```

### Fundamental Reset（FLR）

FLR（Function Level Reset）是 PCIe 规范要求的、不需要断电的 Reset 机制。

源码：`drivers/pci/pci.c:4330`

```c
int pcie_flr(struct pci_dev *dev)
{
    int ret;

    /* 1. 等待所有 pending DMA 和事务完成 */
    if (!pci_wait_for_pending_transaction(dev))
        pci_err(dev, "timed out waiting for pending transaction\n");

    /* 2. 停止 IOMMU（如果有 ATS/SVA）*/
    ret = pci_dev_reset_iommu_prepare(dev);
    if (ret)
        return ret;

    /* 3. 发起 FLR（向 Device Control Register 写 FLR 位）*/
    pcie_capability_set_word(dev, PCI_EXP_DEVCTL, PCI_EXP_DEVCTL_BCR_FLR);

    if (dev->imm_ready)
        goto done;  /* 设备支持 immediate ready，跳过等待 */

    /* 4. PCIe r4.0 规范：FLR 最多 100ms 完成 */
    msleep(100);

    /* 5. 等待设备恢复（pci_dev_wait 最多 1000ms）*/
    ret = pci_dev_wait(dev, "FLR", PCIE_RESET_READY_POLL_MS);

done:
    pci_dev_reset_iommu_done(dev);
    return ret;
}
```

**FLR 特点**：
- 不需要断电，设备保留配置空间内容
- 只能 Reset 单个 Function（一个 PCIe 设备可能有多个 Function）
- 100ms 内必须完成

### DPC（Downstream Port Containment）

DPC 是 RC 或者 Switch 下游端口的错误Containment机制，检测到错误后自动关闭下游端口。

源码：`drivers/pci/probe.c:2675`

```c
pci_dpc_init(dev);  /* Downstream Port Containment 初始化 */
```

## B. 决策树

```
设备电源问题
  ├── D3hot 无法唤醒 → 检查 PME Capability
  ├── 单 Function Reset → pcie_flr()（写 FLR 位）
  ├── 链路错误后恢复 → DPC（硬件自动触发）
  └── 完全断电恢复 → D3cold → 重新枚举

pcie_flr() 执行流程：
  1. pci_wait_for_pending_transaction()  ← 等待 pending DMA
  2. pci_dev_reset_iommu_prepare()      ← 停止 IOMMU
  3. 写 PCI_EXP_DEVCTL_BCR_FLR          ← 发起 FLR
  4. msleep(100)                         ← 等 FLR 完成
  5. pci_dev_wait()                      ← 等设备重新响应
```

## C. 常见错误

| 错误现象 | 原因 | 解决方法 |
|---------|------|---------|
| D3hot 设备无法唤醒 | PME 未使能 | 驱动里调用 `device_init_wakeup(dev, true)` |
| FLR 后设备不响应 | 设备不支持 FLR | 换用传统 Reset（关闭电源再开）|
| D3cold 后配置丢失 | D3cold 完全断电 | 驱动需要重新 probe |
